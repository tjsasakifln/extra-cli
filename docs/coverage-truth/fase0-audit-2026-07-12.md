## 1. Definição Canônica do Universo (Regra #1)

| Conceito | Valor | Fonte | Notas |
|----------|-------|-------|-------|
| Total seed rows | **2.085** | Planilha "Entes Públicos SC" + `EXPECTED_COUNT = 2085` | Fonte canônica |
| Com coordenadas | **1.481** | Planilha (lat/lon não-NULL) | 1.481 = 2.085 - 604 |
| Sem coordenadas | **604** | Planilha (lat/lon NULL) | Bloco não-resolvido |
| **Raio 200km (UNIVERSO CANÔNICO)** | **1.093** | Coluna "Raio 200km?" = SIM + Haversine ≤ 200km | **ESTE é o denominador para cobertura** |
| Fora do raio | **388** | 1.481 - 1.093 | Subconjunto dos com coordenadas |
| Denominador conservativo | **1.697** | 1.093 + 604 | Usado quando não-resolvidos são incluídos |
| CNPJ8 únicos no raio | **~1.000** | target_universe.py (pós-dedup) | ~80 duplicatas de CNPJ8 |

**Identificação:** `entity_id` (PK `sc_public_entities.id`) + `cnpj_8` (CNPJ raiz 8 dígitos, UNIQUE INDEX). Matching: `LEFT(orgao_cnpj, 8) = cnpj_8`.

**Decisão:** O universo canônico para TODOS os módulos é **1.093 entes** dentro do raio 200km, identificados por `entity_id` com `cnpj_8` normalizado. Este número DEVE ser o mesmo em todos os manifests, métricas, e testes.

### 2.2 Métricas Comerciais — Transaction Aborted

**Arquivo:** `output/readiness/coverage_manifest.json`, seção `commercial_metrics`

```
contract_total_value → ERROR: "canceling statement due to statement timeout"
desagio              → ERROR: "current transaction is aborted, commands ignored until end of transaction block"
relicitacao_probability → ERROR: "current transaction is aborted, commands ignored until end of transaction block"
win_rate             → "manual" (não calculado)
```

**Root cause:** `consulting_readiness.py` usa `conn.autocommit = True`, mas após statement timeout, PostgreSQL rejeita qualquer comando na mesma conexão até ROLLBACK explícito. Nenhum ROLLBACK é emitido. As queries 2 e 3 herdam a transação abortada.

### Regra #3 — success_zero exige fonte aplicável, janela, paginação completa, execução real
