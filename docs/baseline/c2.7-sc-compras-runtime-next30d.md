## Bug fix aplicado (mínimo)

**Sintoma (1ª tentativa):**  
`Upsert failed: new row for relation "pncp_raw_bids" violates check constraint "chk_pncp_raw_bids_esfera_id"`

**Causa:** crawler emitia `esfera_id` letra (`E`/`M`); constraint pós-migration 049 aceita apenas `'1'|'2'|'3'|'4'`.

**Fix:** mapear em `_normalize_item`:

| Letra | PNCP `esfera_id` |
|-------|------------------|
| F | `1` (Federal) |
| E | `2` (Estadual) |
| M | `3` (Municipal) |

Arquivos:

- `scripts/crawl/sc_compras_crawler.py` — mapa `_ESFERA_LETTER_TO_ID`
- `tests/test_sc_compras_crawler.py` — asserts de transform atualizados

**Testes:** `pytest tests/test_sc_compras_crawler.py` → **64 passed**, 24 skipped.
