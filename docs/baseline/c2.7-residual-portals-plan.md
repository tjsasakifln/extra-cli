## 4. Public vs blocked — o que pode rodar agora

| Classe | Sources | Ação permitida já | Ação bloqueada |
|--------|---------|-------------------|----------------|
| **PUBLIC RUN** | `sc_compras`, `tce_sc`, `ciga_ckan`, `pcp`, `compras_gov`, `pncp` | dry-run, incremental, full (respeitar rate limits); persistir se DSN ok | Claims de 95% sem reconciliation |
| **PUBLIC FRAGILE** | `transparencia` (HTTP templates), residual scrape | Smoke de import; sample 3–5 municípios HTTP-only (`requires_js: false`) | Batch 220 sem budget de tempo/Selenium |
| **BLOCKED AUTH** | `doe_sc`, `dom_sc` | Import module; `crawl()` → `[]`; validate credentials = False | Monitor mode real com expectativa de dados |
| **CONDITIONAL** | `mides_bigquery` | Validator path ok | Query BQ sem provar SA + dataset |
| **STALE / OUT** | `doe_sc_selenium` | Não priorizar | Investir em Selenium DOE se API auth resolver |

### Decisão C2.7

- **Sprint imediato (sem secrets):** `sc_compras` + smoke `tce_sc` + amostra transparência HTTP.  
- **Sprint gated:** `doe_sc` assim que `DOE_SC_*` preenchidos na VPS/local.  
- **Sprint de engenharia pesada:** famílias Betha/residual (Playwright ou API reverse).  
- **Não reabrir:** path autenticado DOM como crítico path (CIGA já canônico).

### Fase 0 — Preparação (este doc) ✅

- [x] Inventory registry residual

### Fase 1 — ROI estadual público (D+1 a D+2)

1. **`sc_compras` full/list-only** → persist `source=sc_compras` (ou alias monitor)  
2. Contagem por órgão + join attempt com `sc_public_entities` / universo 200 km  
3. **`tce_sc` incremental** (órgão TCE) + evidência contracts capability  
4. Atualizar coverage truth / `ingestion_runs` — sair de `never`

### Fase 2 — DOE-SC gated (D+1 após secret)

1. Preencher secrets **somente** em secret store / `.env` local (não git)  
2. `validate_source_credentials("doe_sc")` → ok  
3. `crawl("incremental")` janela 1d; se 401 → cred inválida  
4. Full 90d quando incremental estável; desligar expectativa em selenium DOE  

### Fase 3 — Transparência seletiva (paralelo se capacidade)

1. Rodar **apenas** municípios `requires_js: false`  
2. Não batchar 59 Betha sem solução JS  
3. Residual: classificar top-N do **raio 200 km** (não os 231 de SC inteiro primeiro)  
4. Documentar inviáveis com código (`offline`, `captcha`, …) em artefato atualizado  

### Fase 4 — Condicional / defer

| Item | Quando atacar | Quando adiar |
|------|---------------|--------------|
| MIDES BQ | Se gap de histórico estadual/municipal crítico | Se PNCP+CIGA+SC Compras bastarem para open_tenders |
| DOM auth | Nunca como path crítico | Só auditoria se CIGA falhar |
| Fiorilli/etc. | Só se detecção nova achar hits | Default: ignore |
| Selenium DOE | Só se API DOE permanentemente blocked com cred válida | Preferir API |

### Critério de “C2.7 done suficiente” (operacional, não DoD 95%)

- [ ] `sc_compras`: ≥1 run fresco com `n_records > 0` em DB + report
