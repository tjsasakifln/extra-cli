## 2. Comportamento de resume / watermark

### 2.1 `pncp_backfill.py` (página-atômico — C2.3)

1. Valida janela **exatamente 7 dias inclusivos** e **fechada** (`date_to < hoje`).
2. Sem `--resume`: cria `pncp_backfill_runs` (`status=running`, `run_id=pncp-{from}-{to}-{uuid12}`).
3. Com `--resume`: localiza último run da mesma janela com status `running|interrupted|failed` e reabre.
4. Para cada `modalidade_id` em `DEFAULT_MODALIDADES`:
   - lê páginas já em `pncp_backfill_pages` → `SKIP ... checkpoint=completed`;
   - fetch página → transform → `upsert_pncp_raw_bids` + insert membership + insert page **na mesma transação** → `COMMIT`;
   - se SIGINT/SIGTERM: marca `interrupted` **após** a página atual (nunca no meio do commit).
5. Falha de fetch/paginação/totalPaginas: `failed` + `error_message`.
6. Sucesso: exige todas as modalidades e `pages_completed == pages_expected` → `completed`.
7. `--reconcile`: re-lê a API de forma independente e compara IDs/contagens do run.

**Não há watermark global de data multi-janela no 049.** Cada janela de 7 dias é um run independente. Orquestrar 90d/3y = **loop externo de janelas**.

### 2.2 Contratos `backfill_3y` (arquivo)

1. Calcula `start = today - CONTRACTS_BACKFILL_YEARS*365` → `today`.
2. Particiona em janelas de `CONTRACTS_WINDOW_DAYS` (30).
3. Checkpoint JSON: `completed_windows[]`, `current_window_start`, contadores, `last_error`.
4. Skip se `window_key` já em `completed_windows`.
5. **Limitação conhecida (plano-mestre / DoD):** se a janela falhar **depois** de obter dados parciais, pode ser marcada completed quando `window_records > 0` mesmo com erros de páginas posteriores → risco de **cobertura parcial silenciosa**. Preferir `crawl_with_evidence()` + inspeção de evidence em runs sérios.
6. Upsert: RPC `upsert_pncp_supplier_contracts` (migration 006). Há relato de `DO NOTHING` legado em alguns caminhos — validar que alterações de valor/vigência são refletidas (K3.4).

### 2.3 Adapter + `pipeline_watermarks`

- `pncp_crawler_adapter.crawl_contracts(..., resume=True)` lê watermark `source=pncp_contracts`, `scope_key=page`.
- Commit por página via `commit_watermark` (046).
- **Nota:** watermark de página **sem** chave de janela de data no scope default pode ser ambíguo multi-janela; o caminho operacional preferido para 3y de contratos permanece o checkpoint JSON do `contracts_crawler`.

Loop controlado 90d (12× janelas de 7d; **não** rodar em foreground sem `tmux`/`systemd`):

```bash
python3 - <<'PY'
from datetime import date, timedelta
from pathlib import Path
import subprocess, sys, os

dsn = os.environ["DSN"]  # exporte DSN antes
end = date.today() - timedelta(days=1)
# 12 janelas * 7 = 84 dias fechados (resto <7d fora do contrato 049)
n_windows = 12
start = end - timedelta(days=n_windows * 7 - 1)
cur = start
while cur + timedelta(days=6) <= end:
    a, b = cur, cur + timedelta(days=6)
    cmd = [
        sys.executable, "-m", "scripts.crawl.pncp_backfill", "run",
        "--dsn", dsn,
        "--date-from", a.isoformat(),
        "--date-to", b.isoformat(),
        "--request-delay", "1.1",
        "--resume",  # no-op se não houver run incompleto → falha; ver nota
    ]
    # Estratégia: tenta resume; se ERROR "nenhuma execucao incompleta", roda sem --resume
    print("WINDOW", a, b, flush=True)
    r = subprocess.run(cmd)
    if r.returncode != 0:
        cmd2 = [c for c in cmd if c != "--resume"]
        r2 = subprocess.run(cmd2)
        if r2.returncode not in (0, 130):
            raise SystemExit(r2.returncode)
        if r2.returncode == 130:
            print("interrupted", a, b); raise SystemExit(130)
    cur = b + timedelta(days=1)
print("PILOT_90D_PUBLICACOES_DONE")
PY
```

**Nota CLI:** `--resume` **falha** se não existir run incompleto; o loop acima faz fallback para run novo.

       pages_completed, pages_expected, records_fetched,

## 5. Riscos

| ID | Risco | Severidade | Mitigação |
|----|-------|------------|-----------|
| R-TO | **Timeout / API lenta** (C2.3 já viu timeout 90s em `crawl('incremental')`) | Alta | timeouts configuráveis; retries; janelas menores; não confundir timeout com bug de código |
| R-429 | **Rate limit HTTP 429** (histórico: delay 0.3s insuficiente; 1.0–1.5s mais seguro) | Alta | `--request-delay ≥ 1.1`; `CONTRACTS_REQUEST_DELAY`/`JANELA_DELAY`; backoff + Retry-After no adapter |
| R-PAGE | Publicações: **>10k registros** em modalidade×7d (`50×PNCP_MAX_PAGES`) → truncamento | Alta | monitorar `totalPaginas`; aumentar `PNCP_MAX_PAGES` só com cuidado; reduzir janela se API permitir |
| R-MODE | `monitor --mode backfill` **≠** `backfill_3y` | Alta | documentado; usar `mode='backfill_3y'` via API |
| R-PARTIAL | Checkpoint contratos marca janela completed com dados parciais | Alta | `crawl_with_evidence`; não confiar só em `completed_windows` |
| R-UPSERT | Upsert contratos pode não atualizar mudanças (DO NOTHING legado) | Média | validar amostra K3.1/K3.4; reprocess controlado |
| R-UF | `/contratos` no crawler atual **sem filtro UF** → volume BR inteiro | Alta | disco/tempo; filtrar pós-ingest por SC/raio; avaliar filtro API se existir |
| R-049-SCOPE | 049 hardcode **SC** e janela **7d** | Média | loop externo; não usar para multi-UF sem alteração de schema |
| R-049-RESUME | `--resume` sem run incompleto → erro | Baixa | fallback run novo no orquestrador |
| R-MIG | DB local sem tabelas 046/049 | Alta | aplicar migrations / fresh path L1 |
| R-DISK | 3y contratos + `raw_payload` em 049 | Média | monitorar tamanho DB; backup pré-backfill |
| R-SIGINT | Kill -9 perde página em voo | Baixa | página atômica no 049; re-run `--resume` |
| R-IDEMP | Segundo run 049 na mesma janela **cria novo run** (sem `--resume`) | Média | upsert por content_hash evita duplicar fatos; membership fica por run |
| R-SCHEMA | Drift `valor_total` vs `valor_global` (K3.1) | Alta p/ analytics | não bloquear ingestão; bloquear claims de valor até K3.4/K3.6 |
