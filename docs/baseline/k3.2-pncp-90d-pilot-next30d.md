## 3. Bloqueios descobertos e correções (não reverter)

### 3.1 Janela parcial silenciosa (já no HEAD de trabalho)

Em `_crawl_date_range` / `crawl_with_evidence`:

- **Antes:** janela com erros de página mas `records > 0` podia ir para `completed_windows`.
- **Agora:** `fully_ok = not window_errors` / `window_status.is_success` — partial **não** completa.
- Testes: `tests/test_contracts_window_complete.py` (3 passed).

**Prova ao vivo:** janela `20260417_20260516` page 13 →  
`connection_failed … read operation timed out` → log  
`Window … NOT marked complete (errors=1, transformed=6000)`.

### 3.2 RPC upsert ambíguo (migration 050)

`upsert_pncp_supplier_contracts` retornava `TABLE(..., contrato_id text)` e usava `ON CONFLICT (contrato_id)` / `RETURNING contrato_id` → **`AmbiguousColumn`**.

**Fix:** colunas de entrada `in_contrato_id`, `ON CONFLICT ON CONSTRAINT pncp_supplier_contracts_contrato_id_key`, `DISTINCT ON (contrato_id)`.

Arquivo: `db/migrations/050_fix_contracts_upsert_fk.sql` (aplicada no DB local `pncp_datalake`).

### 3.3 FKs SC-only bloqueavam ingestão nacional

| FK | Problema |
|----|----------|
| `fk_contracts_orgao_entity_v2` | `orgao_cnpj_8` → `sc_public_entities` (~2085 rows SC) |
| `fk_contracts_supplier_entity_v2` | fornecedor ≠ entidade pública; hit-rate **0/500** na amostra |

**Fix (050):** `DROP CONSTRAINT` das duas FKs. Reintroduzir só com universo de entidades nacional (fora do escopo K3.2 pilot).

### 3.4 Checkpoint `full`

Checkpoint reentrante passou a valer para `mode in {full, backfill_3y}` (antes só `backfill_3y`).  
`SOURCE_PURPOSE = "contracts"` exportado no módulo.
