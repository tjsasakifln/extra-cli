# ADR-005: GPT-4.1-nano para Classificação de Editais

**Status:** Aceito
**Data:** 2026-07-10
**Decisor:** Tiago Sasaki
**Fonte:** `config/settings.py`, `intel_llm_gate.py`, `sectors_config.yaml`

---

## Contexto

O pipeline de inteligência precisa classificar se um edital é relevante para a Extra Construtora. A classificação puramente heurística (keywords) gera falsos positivos. Um LLM poderia entender o contexto do objeto da licitação.

## Decisão

**Usar GPT-4.1-nano como classificador binário (SIM/NAO) com fallback seguro (REJECT). Heurísticas como primeira camada, LLM como segunda.**

## Justificativa

- GPT-4.1-nano é o modelo mais barato da OpenAI (~$0.075/1M tokens input)
- Classificação binária com prompt curto → custo por edital ≈ $0.0001
- Timeout de 10s evita bloqueios no pipeline
- Estratégia zero-noise: erro na API = REJECT (não deixa passar ruído)
- Heurísticas (Camada 1) filtram a maioria, LLM só nos casos ambíguos

## Consequências

- ✅ Custo baixíssimo por classificação
- ✅ Zero falsos positivos (REJECT on failure)
- ✅ Heurísticas cobrem 80%+ dos casos sem chamar API
- ❌ Dependência de API externa (OpenAI)
- ❌ Timeout de 10s pode ser insuficiente para editais muito longos
- ❌ GPT-4.1-nano pode ser descontinuado (modelo experimental)
